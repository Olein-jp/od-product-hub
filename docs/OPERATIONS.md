# 運用ダッシュボードとログ

「OD Product Hub」トップでは、有効・停止ライセンス、支払い失敗、今月の新規契約、Webhookエラーの件数と、直近のWebhook/APIログを確認できます。各集計カードとログ行から関連画面へ移動できます。

「OD Product Hub → ログ」にはWebhook、API、管理操作のタブがあります。各一覧は20件単位でページングし、条件に一致する行だけをDBから取得します。WebhookはイベントID・種類・結果、APIは操作・結果・エラーコード・サイトURL、管理操作は操作・対象種別・ユーザーIDで絞り込めます。

Webhookログには試行回数、最終試行日時、重複配送数、最終エラーを表示します。Handlerが失敗したイベントはStripeの再送ごとに原子的に再claimされ、同時に届いた別リクエストは処理を横取りしません。`processing` が5分以上残った場合は停止した処理とみなして再claimし、5回失敗すると `exhausted` になります。

`error` / `exhausted` の詳細画面では「Stripeからの再試行を許可」を実行できます。この操作は `manage_options` とnonceで保護され、管理操作ログへ記録されます。保存済みpayloadは個人情報をマスク済みで完全なStripeイベントではないため、画面から直接再生しません。操作後にStripe DashboardまたはStripe CLIから元イベントを再送してください。再試行前にDB、商品対応、Stripe IDなど最初の失敗原因を解消してください。

Webhook payloadは保存前と詳細表示時の両方で、メールアドレス、氏名、住所、電話番号、カード・銀行口座・支払い関連情報、secretをマスクします。無効なJSONは内容を表示せず、無効だったことだけを記録します。

Webhook、API、管理操作、メールログは、設定画面の「ログ保持日数」（既定365日）より古くなると日次WP-Cronで削除されます。プラグイン有効化時に初回実行を1時間後として登録し、以後は通常の初期化時にも登録状態を確認します。このため、更新時にイベントが未登録でも自動的に1件だけ復旧します。登録に失敗した場合は初期化を例外で停止し、Cronが動作するように見せかけたまま処理を継続しません。プラグイン無効化時にはイベントを削除します。

同じ処理はログ画面から手動実行でき、固定件数のバッチ処理なので繰り返し実行しても安全です。低アクセス環境で実行時刻を保証する必要がある場合は、サーバーCronからWordPress Cronを定期実行してください。

統合確認は次のコマンドで実行します。

```sh
npm run test:operations
```

## 個人データ要求への対応

WordPress管理画面の「ツール → 個人データのエクスポート」「ツール → 個人データの消去」から、本人確認済みのメールアドレスに対してOD Product Hubのデータを処理できます。同じメールアドレスを持つ顧客レコードが複数ある場合もすべて対象とし、50件ずつ処理します。

消去時はAPI・ダウンロードログのサイトURL、IPアドレス、User-Agentを匿名化します。顧客・契約・ライセンス・メール配信記録は契約履行、会計、不正利用防止、Stripe同期のため保持され、結果画面に保持理由が表示されます。Stripe側の顧客・契約・決済情報はWordPressから消去されないため、必要に応じてStripe側でも別途対応してください。

マルチサイトでは各サイトのテーブル接頭辞に従って、そのサイトのOD Product Hubデータだけを処理します。ネットワーク全体を一括処理しないため、複数サイトでプラグインを利用している場合は各サイトの管理画面で要求を実行してください。

統合確認は `npm run test:privacy` で実行します。複数顧客、該当なし、再実行、50件超のページング、機密情報の非出力、ログ匿名化、契約データ保持を検証します。

## Site Health診断

「ツール → サイトヘルス」では、本番依存を5つの独立した診断として確認できます。

- Stripe/HTTPS: 3つのStripe設定、HTTPS（`local` / `development` は例外）、手動接続試験の最終結果と成功日時
- Webhook: 最終成功日時、3件以上の連続失敗、5分を超えた `processing`
- ログ保持: `odph_cleanup_logs` の登録、次回実行、最終成功日時
- 更新配布: Sodium/Zip、秘密ストレージの書込可否とWebルート外配置、公開リリースのファイル・SHA-256・Ed25519署名
- 購入・マイページ: 完了／キャンセルURLと対応ショートコード、公開済みマイページと `[odph_my_account]`

Stripeへの外部通信はSite Health表示時には行いません。「OD Product Hub → 設定」の手動接続試験を実行すると、その結果だけを診断へ保存します。異常時は診断内のリンクから設定またはログ画面へ移動し、修正後にSite Healthを再実行してください。コピー可能なデバッグ情報には設定済みかどうかと安全な日時だけを含め、Secret、顧客情報、ライセンスキー、実ファイルパスは含めません。

診断の正常・異常・回復と情報非露出は `npm run test:site-health` で検証します。

このテストは集計値、検索、payloadマスキング、1万件時のページング、保持期間の変更、全ログ種別の削除に加え、既存インストールで欠けたCronの自己修復、重複防止、無効化時の削除を検証します。Webhookの失敗回復、並行再送、stale processing、再試行上限は `npm run test:webhook` で検証します。
